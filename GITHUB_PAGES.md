# Инструкция по настройке GitHub Pages

Это руководство поможет вам настроить GitHub Pages для размещения лендинга ChatList.

## Способ 1: Использование папки docs (Рекомендуется)

### Шаг 1: Подготовьте HTML-лендинг

1. Создайте папку `docs` в корне проекта (если её нет)
2. Поместите файл `index.html` в папку `docs`
3. (Опционально) Добавьте файлы стилей и изображений в `docs`

### Шаг 2: Настройте GitHub Pages

1. **Откройте настройки репозитория:**
   - Перейдите на страницу вашего репозитория на GitHub
   - Нажмите "Settings" (Настройки)

2. **Найдите раздел Pages:**
   - В левом меню найдите раздел "Pages"
   - Или перейдите по адресу: `https://github.com/ваш-username/ChatList/settings/pages`

3. **Настройте источник:**
   - В разделе "Source" выберите "Deploy from a branch"
   - Выберите ветку: `main` (или `master`)
   - Выберите папку: `/docs`
   - Нажмите "Save"

4. **Дождитесь публикации:**
   - GitHub автоматически опубликует сайт
   - Обычно это занимает 1-2 минуты
   - После публикации сайт будет доступен по адресу: `https://ваш-username.github.io/ChatList/`

### Шаг 3: Проверьте работу

1. Откройте адрес вашего сайта: `https://ваш-username.github.io/ChatList/`
2. Убедитесь, что лендинг отображается корректно
3. Проверьте все ссылки и изображения

## Способ 2: Использование ветки gh-pages

Если вы предпочитаете использовать отдельную ветку:

```powershell
# Создайте ветку gh-pages
git checkout -b gh-pages

# Скопируйте файлы лендинга в корень ветки
# (или переместите содержимое папки docs в корень)

# Закоммитьте изменения
git add .
git commit -m "Add GitHub Pages landing page"

# Отправьте ветку на GitHub
git push origin gh-pages
```

Затем в настройках Pages выберите ветку `gh-pages` и папку `/ (root)`.

## Обновление лендинга

После изменения файлов в папке `docs`:

```powershell
# Добавьте изменения
git add docs/

# Закоммитьте
git commit -m "Update landing page"

# Отправьте на GitHub
git push origin main
```

GitHub автоматически обновит сайт через 1-2 минуты.

## Настройка кастомного домена (Опционально)

Если у вас есть собственный домен:

1. В настройках Pages добавьте ваш домен в поле "Custom domain"
2. Настройте DNS записи у вашего провайдера домена:
   - Добавьте CNAME запись, указывающую на `ваш-username.github.io`
3. Дождитесь применения изменений DNS (может занять до 24 часов)

## Структура файлов

Рекомендуемая структура для GitHub Pages:

```
ChatList/
├── docs/
│   ├── index.html          # Главная страница лендинга
│   ├── css/
│   │   └── style.css      # Стили (опционально)
│   ├── images/
│   │   └── screenshot.png  # Скриншоты (опционально)
│   └── assets/            # Другие ресурсы
├── .github/
│   └── workflows/
│       └── pages.yml       # Автоматическая публикация (опционально)
└── ...
```

## Автоматизация через GitHub Actions

Для автоматической публикации при каждом коммите создайте файл `.github/workflows/pages.yml` (см. отдельный файл).

## Рекомендации

1. **Используйте относительные пути:** Все ссылки на ресурсы должны быть относительными
2. **Оптимизируйте изображения:** Используйте сжатые изображения для быстрой загрузки
3. **Тестируйте локально:** Откройте `index.html` в браузере перед публикацией
4. **Проверяйте мобильную версию:** Убедитесь, что сайт корректно отображается на мобильных устройствах
5. **Добавьте favicon:** Добавьте файл `favicon.ico` в папку `docs`

## Устранение проблем

### Сайт не обновляется

- Подождите несколько минут (GitHub может кэшировать страницы)
- Очистите кэш браузера (Ctrl+F5)
- Проверьте, что файлы закоммичены и отправлены на GitHub

### Изображения не отображаются

- Проверьте пути к изображениям (должны быть относительными)
- Убедитесь, что файлы изображений добавлены в репозиторий
- Проверьте права доступа к файлам

### Стили не применяются

- Проверьте пути к CSS файлам
- Убедитесь, что файлы CSS добавлены в репозиторий
- Проверьте синтаксис CSS на ошибки
